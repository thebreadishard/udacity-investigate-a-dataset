Introduction¶

Dataset Description¶

This project analyzes the TMDb movie dataset, which contains information on 10,866 movies collected from The Movie Database (TMDb). The data was cleaned by Udacity from the original Kaggle release. It is a single table with 21 columns:

Column Description
id TMDb identifier for the movie
imdb_id IMDb identifier (e.g. tt0369610)
popularity TMDb popularity score (relative, unbounded)
budget Production budget in nominal US dollars
revenue Box-office revenue in nominal US dollars
original_title Title of the movie
cast Lead actors, pipe-separated (\|), capped at five names
homepage Official website URL
director Director(s) of the movie
tagline Marketing tagline
keywords Plot keywords, pipe-separated (\|)
overview Short plot summary
runtime Length of the movie in minutes
genres Genres, pipe-separated (\|)
production_companies Production companies, pipe-separated (\|)
release_date Release date (stored as text)
vote_count Number of user votes
vote_average Average user rating (0–10)
release_year Year of release (integer)
budget_adj Budget in inflation-adjusted 2010 US dollars
revenue_adj Revenue in inflation-adjusted 2010 US dollars

A few characteristics are important for the analysis. Four columns (cast, keywords, genres, production_companies) store multiple pipe-separated values in a single field. The budget_adj and revenue_adj columns express money in 2010 dollars, which makes financial comparisons across decades fair, so they are preferred over the raw budget / revenue columns. Finally, several numeric columns use 0 to encode missing data (notably budget, revenue, and runtime), which is addressed in the Data Wrangling section.

Question(s) for Analysis¶

This report investigates what is associated with a movie's success, using two complementary outcome measures as the primary dependent variables:

  • revenue_adj — box-office revenue in inflation-adjusted 2010 US dollars, and
  • vote_average — the average TMDb user rating.

These outcomes are explored against several independent variables — listed cast size, release year, actor prolificness, production budget, and genre — through the following four questions:

  1. Cast size over time and success: How has the number of listed cast members changed across release years, and is listed cast size related to a movie's revenue? (independent variables: listed cast size, release_year)
  2. Small-cast performance: Do movies with small listed casts (1–3 names) perform differently from full 5-listed-cast movies on revenue, rating, and popularity? (independent variable: listed cast-size group)
  3. Veteran stars: Do movies featuring a prolific/veteran actor — measured by the number of films an actor appears in within this dataset — earn more or score higher than movies without one? (independent variables: presence of a prolific actor, budget_adj)
  4. Genre landscape: Which genres are associated with higher ratings versus higher revenue, and where do niche illness/"dementia" themes sit within that landscape? (independent variable: genres)

The analysis uses a mix of single-variable (1d) and multiple-variable (2d) views, and each question carries a clearly stated limitation (for example, the cast column is capped at five names, and film count is only a proxy for an actor's seniority).

Data Cleaning¶

Based on the inspection above, the cleaning plan is:

  1. Remove the single fully duplicated row.
  2. Treat zeros as missing. budget, revenue, budget_adj, revenue_adj, and runtime use 0 to encode missing measurements. We convert those zeros to NaN so that financial/runtime analyses can drop them per question rather than deleting whole rows up front (which would needlessly discard movies that still have valid ratings, genres, and cast information).
  3. Leave descriptive text columns as-is. Missing values in homepage, tagline, keywords, etc. are not central to the questions, so no rows are dropped for them.

The steps are implemented as small, reusable functions so the same logic can be applied to other datasets that share these common problems (duplicate rows and zero-coded missing values).

# --- Reusable, dataset-agnostic data-cleaning helpers ---
# These functions take any DataFrame so the same cleaning logic can be reused on
# other datasets, not just this one.

def drop_full_duplicates(data, verbose=True):
    """Return a copy of `data` with exact duplicate rows removed and the index reset."""
    n_before = len(data)
    cleaned = data.drop_duplicates().reset_index(drop=True)
    if verbose:
        print(f'Removed {n_before - len(cleaned)} duplicate row(s); {len(cleaned)} rows remain.')
    return cleaned


def zeros_to_nan(data, columns, verbose=True):
    """Return a copy of `data` where 0 values in `columns` are treated as missing (NaN).

    Useful whenever a dataset encodes missing measurements as 0 instead of NaN.
    """
    cleaned = data.copy()
    for col in columns:
        n_zeros = int((cleaned[col] == 0).sum())
        cleaned[col] = cleaned[col].replace(0, np.nan)
        if verbose:
            print(f'{col}: converted {n_zeros} zero value(s) to NaN.')
    return cleaned

Exploratory Data Analysis¶

Shared analysis helpers¶

Before exploring the individual questions, we define a small set of reusable helper functions used throughout the EDA: count_pipe_items (to count pipe-separated values such as cast or genres) and style_axes (to apply a consistent title and axis labels to every plot). Defining them once keeps the analysis DRY and ensures all figures are labelled the same way.

In [10]:
# --- Reusable helpers used across all of the analysis questions ---

def count_pipe_items(series):
    """Number of pipe-separated ('|') items in each value of a text Series.

    Missing values count as 0. Generic for any '|'-delimited column
    (cast, genres, keywords, production_companies, ...).
    """
    return series.fillna('').apply(lambda s: sum(1 for item in s.split('|') if item))


def pipe_value_counts(series):
    """Global frequency of each pipe-separated ('|') token across a text Series.

    Generic for any '|'-delimited column: returns how often each individual
    token (actor, genre, ...) appears across all rows.
    """
    exploded = series.fillna('').str.split('|').explode()
    exploded = exploded[exploded.str.strip() != '']
    return exploded.value_counts()


def style_axes(ax, title, xlabel, ylabel):
    """Apply a consistent title and axis labels to a matplotlib Axes and return it."""
    ax.set_title(title)
    ax.set_xlabel(xlabel)
    ax.set_ylabel(ylabel)
    return ax

Findings — Research Question 1

  • Distribution (1d): Listed cast size is heavily concentrated at the maximum of 5 (10,133 of 10,865 movies, ~93%). Only 656 movies list 1–4 names and 76 list none, giving a mean of 4.84 and a median of 5. The 76 movies that list no cast are overwhelmingly documentaries (42) and animation (30) — genres where a fictional cast is often absent or simply not entered, so the 0 reflects a real characteristic of those films rather than data to be "fixed".
  • Over time (2d): The mean listed cast size does not rise over the years; if anything it drifts slightly down, from ~5.0 in the earliest years to ~4.7 by 2010–2015. The completeness table above shows this is an artifact of cataloguing, not a real trend: the number of movies in the dataset explodes from ~362 across the entire 1960s to ~3,116 in 2011–2015, while the share of titles listing fewer than 5 names climbs from ≈0% (virtually every older film lists a full 5) to ~11% in recent years. Recent years sweep in a long tail of small/obscure titles whose casts are only partially entered, dragging the mean down — this is not a change in how many actors films actually employ.
  • Vs. revenue (2d): Movies that list a full cast of 5 have a far higher mean inflation-adjusted revenue (~117M USD) than those listing fewer names (≤ 8M USD), yet the overall correlation between cast_count and revenue_adj is essentially zero (r ≈ 0.06). The gap is therefore driven by catalogue completeness — popular, high-grossing films get fully listed — not by cast size itself.

Limitation: because cast is capped at five names, cast_count measures listing completeness rather than true ensemble size, so it cannot tell us whether bigger real casts relate to success. We should not read the revenue gap as cast size causing higher revenue.

Research Question 2: Small-cast vs full-cast performance¶

RQ1 suggested that listed cast size is largely a measure of cataloguing completeness rather than true ensemble size. Here we test that idea directly by grouping movies by listed cast size and comparing how each group performs on three success measures: revenue (revenue_adj), rating (vote_average), and popularity (popularity).

  • 1d: how many movies fall into each cast-size group.
  • 2d: how revenue and rating differ across those groups.

We bucket movies into 0 (none), 1–3 (small), 4, and 5 (full) listed cast members, and also track vote_count and how often revenue is even recorded — so we can tell genuine performance apart from the obscurity/cataloguing artifact.

In [16]:
# Group movies by listed cast size so we can compare 'small' vs 'full' casts.
def label_cast_group(count):
    """Bucket a listed-cast count into an ordered, human-readable group."""
    if count == 0:
        return '0 (none)'
    if count <= 3:
        return '1-3 (small)'
    if count == 4:
        return '4'
    return '5 (full)'


group_order = ['0 (none)', '1-3 (small)', '4', '5 (full)']
df_clean['cast_size_group'] = pd.Categorical(
    df_clean['cast_count'].apply(label_cast_group),
    categories=group_order, ordered=True)

# Compare the success measures across groups. Revenue is highly skewed, so we use the
# median; we also track vote_count and how often revenue is even recorded, to expose
# whether differences reflect performance or simply catalogue completeness.

Findings — Research Question 2

Comparing movies by listed cast-size group (the table and plots above):

  • Commercially, full-cast movies dominate. Movies listing all 5 names have a median inflation-adjusted revenue of ~45.6M USD, versus only ~0.8M–2.1M USD for the small (1–3) and partial (4) groups — a 20–50× gap. They are also ~3× more popular (mean popularity 0.68 vs ~0.2) and attract far more votes (median 42 vs ~16).
  • But this is largely the same cataloguing artifact seen in RQ1. Revenue is even recorded for only 7–16% of small/partial-cast movies, versus 47% of full-cast movies. Small-cast titles are overwhelmingly obscure films that were never fully catalogued, so the commercial "gap" mostly reflects which movies are well documented — not a causal effect of cast size.
  • On ratings the pattern slightly flips. Small-cast (1–3) and no-cast movies have a higher mean vote_average (~6.7–6.9) than full-cast movies (~6.0). However, those ratings rest on very few votes (median ~16 vs 42), so they are noisier and likely reflect a small, self-selecting fan audience rather than broad acclaim.

Limitation: as in RQ1, cast is capped at five names, so cast-size group is a proxy for cataloguing completeness / obscurity, not true ensemble size. The revenue comparison also excludes the majority of small-cast movies (no revenue recorded), and the rating comparison rests on sharply unequal vote counts. These differences should not be read as cast size causing commercial or critical success.

Research Question 3: Veteran stars and success¶

Do movies featuring a prolific/veteran actor earn more or score higher? We measure an actor's seniority by how many films they appear in within this dataset, flag each movie for whether it lists at least one such actor, and compare the groups — while keeping an eye on budget_adj, since star actors tend to be cast in bigger-budget films.

  • 1d: the distribution of how many films actors appear in (to choose a "veteran" cutoff).
  • 2d: revenue and rating for movies with vs without a veteran actor, and — to test for a budget confound — median revenue within budget quartiles.

Findings — Research Question 3

Defining a veteran/prolific actor as one appearing in ≥ 20 films in this dataset (379 of 19,026 actors — the top ~2%), about 54% of movies list at least one.

  • Headline gap: movies with a veteran actor have ~2.7× the median inflation-adjusted revenue (~57.8M vs ~21.1M USD) and a slightly higher mean rating (6.04 vs 5.90). They also draw far more votes (median 74 vs 23).
  • …but budget explains almost all of it. Veteran movies also carry ~3.7× the median budget (~31M vs ~8.3M USD). When movies are split into budget quartiles, the revenue gap essentially disappears — within each tier the medians are nearly identical (e.g. Q3 ≈ 80M USD for both groups), with veteran films leading only in the top budget quartile. Veteran actors are mostly cast in bigger-budget productions, and it is the budget that tracks revenue, not star seniority itself.
  • Ratings barely move: mean vote_average differs by only ~0.14 points, so a veteran presence is not associated with meaningfully better reviews.

Limitation: "veteran" here is a within-dataset film count — a rough proxy for an actor's true career experience or stature (the dataset is only a sample, the cast list is capped at five names, and a prolific bit-part actor counts the same as a lead). Because veteran presence is strongly confounded with budget, the raw revenue gap must not be read as veteran actors causing higher revenue.

Findings — Research Question 4

Exploding the pipe-separated genres field gives 20 genres. Mapping each genre by its mean rating and median revenue shows that critical acclaim and commercial success pull in different directions:

  • Prestige, low-grossing genres: Documentary has the highest mean rating (~6.9) but the lowest median revenue (~1.4M USD); Music, History, War and Drama are similar — well-rated but modest earners.
  • Blockbuster, lower-rated genres: Adventure, Action, Science Fiction and Fantasy earn the most (median ~70–115M USD) yet sit in the middle-to-low rating band (~5.7–5.9). Horror is the lowest-rated of all (~5.3).
  • Sweet spot: Animation and Family are the rare genres that both score and earn highly (Animation ~6.4 rating, ~149M USD median revenue).
  • Where the niche illness theme sits: the 18 "dementia / Alzheimer"-themed movies (found via keyword/overview text) land squarely in the prestige-drama quadrant — higher mean rating (~6.5 vs ~6.0 overall) but lower median revenue (~24M vs ~44M USD), and 13 of 18 are Dramas. They are critically appreciated but commercially small.

Limitation: genres overlap (each movie can list several), so the genre groups are not independent. Revenue medians rest only on movies whose revenue is recorded, and the dementia subset is tiny (n = 18) and identified by a crude keyword search — it is descriptive only, far too small for any statistical inference.

⭐ Standout: The dramatic arc of the 18 dementia-themed films¶

The RQ4 probe surfaced 18 films that touch on dementia / Alzheimer's. As a creative add-on, this section examines each film's overarching dramatic arc against a classic storytelling template:

"In his efforts to reach a meaningful goal, a sympathetic protagonist, largely by his own effort, overcomes almost insurmountable odds and while doing so learns a valuable lesson."

For every film we record two versions of the story:

  • real_story — the plot's dramatic arc as it is publicly known.
  • template_story — that same arc rewritten to follow the template above.

A short deviation note flags films whose true arc resists the template — for example tragedies of loss (Iris, Away from Her) where there is no triumph, or a creature-feature (Deep Blue Sea) where the "lesson" is really a punishment. Keeping both versions side by side lets us see how well each film actually fits the heroic template — the basis for the visualization we'll design next.

Who carries the dementia?¶

The 18 films split by which character the disease belongs to. This turns out to track how well each film fits the heroic template: when the protagonist has dementia, the "by his own effort… learns a lesson" arc strains hardest; when a caregiver drives the story, the template fits more conventionally.

  • A — Protagonist has dementia (the afflicted person is the central, story-driving character): Remember, Iris, Imaginaerum, De Zaak Alzheimer, Harvie Krumpet, Nebraska.

Conclusions¶

This project explored the TMDb movie dataset (10,865 movies after cleaning) to ask what is associated with a movie's success, where success was measured two ways: inflation-adjusted box-office revenue (revenue_adj) and audience rating (vote_average). Four questions were investigated.

RQ1 — Does cast size relate to success, and has it changed over time? The mean number of listed cast members drifted slightly down across release years (from ~5.0 in the earliest years to ~4.7 by 2010–2015), and revenue showed only a very weak positive correlation with cast size (Pearson r ≈ 0.06). A diagnostic by era revealed that this apparent "trend" is largely a cataloguing artifact: almost every older film lists a full five names, whereas the flood of small/obscure titles in recent years are only partially catalogued, so a growing share list fewer than five. The drift therefore reflects how completely TMDb records newer versus older films, not a real change in production. No meaningful link between cast size and revenue was found.

RQ2 — Do small-cast movies perform differently from full-cast movies? Grouping movies by cast size (none / 1–3 / 4 / 5-listed) showed that fully-listed movies have far higher median revenue, popularity and vote counts, while sparsely-listed movies sit at the bottom on those commercial measures (their ratings actually run a little higher, but on very few votes, so the difference is noisy). Given the RQ1 finding, this is best read as a side-effect of cataloguing completeness — well-documented (and typically bigger, more recent) films tend to be the ones with a full cast list — rather than evidence that adding cast members improves a movie.

RQ3 — Are movies with veteran (frequently-cast) stars more successful? Defining a "veteran" as an actor with ≥ 20 films in the dataset (379 actors), movies featuring at least one veteran had higher median revenue and higher median budget. After splitting by budget quartile, the veteran revenue advantage shrank substantially within each tier, indicating the headline gap is largely confounded by budget: veteran-led films tend to be bigger-budget productions. Veteran presence is associated with, but not a clear independent driver of, revenue.

RQ4 — What does the genre landscape look like for ratings versus revenue? Acclaim and box office pull in different directions. Documentary, Music, History, War and Drama earn the highest ratings but the lowest revenue (Documentary ≈ 6.9 rating yet only ≈ 1.4M USD median revenue), while Adventure, Action, Sci-Fi and Fantasy earn the most but rate lower. Animation and Family are the rare genres that score well and earn well. A small thematic probe (18 movies mentioning dementia/Alzheimer's) landed in the prestige-drama quadrant: above-average ratings (≈ 6.5 vs ≈ 6.0) but below-average revenue (≈ 24M vs ≈ 44M USD).

Overall takeaway. Across all four questions, the variables examined are at most weakly and indirectly associated with success, and the strongest-looking patterns dissolve once data-completeness and budget are accounted for. The clearest real signal is that genre choice trades acclaim against earnings — there is no single genre that maximizes both for most films.

Limitations¶

  • No causal claims. All results are correlational and descriptive; no statistical tests or models were run, so none of these associations should be read as causes of success.
  • Missing and zero-coded financials. Budget and revenue were 0 for a large share of rows (treated as missing). Revenue-based comparisons therefore reflect only the subset of films with reported earnings, which skews toward larger studio releases and may bias the medians upward.
  • Cataloguing bias. As RQ1/RQ2 showed, completeness of fields like cast varies systematically by era, which can manufacture apparent trends.
  • Overlapping, tiny subsets. Genres overlap (a film counts in several), and thematic subsets like the dementia probe (n = 18) are far too small for inference.

Further research¶

Stronger conclusions would require: controlled comparisons (e.g., regression holding budget, year and genre constant), a financially complete dataset, and proper statistical testing. Profit (revenue − budget) and return-on-budget would also be more meaningful success measures than raw revenue.
